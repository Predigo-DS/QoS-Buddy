🔴 1. Scaler Fitted Per-Segment but Feature Columns Shared CRITICAL

You call preprocess_segment() with fit=False for each segment, but inside the function a new RobustScaler() is always created and fitted on that segment's training portion. The scaler parameter you pass in is completely ignored — it's never used anywhere in the function body.

Impact: Each segment gets its own scaler, which is actually fine for per-segment processing. But if you later need to apply the scaler to new data at inference, you've only saved the last one. More critically, the fit parameter only controls the LabelEncoder — the scaler always re-fits, which could cause train/val/test leakage if you ever change the split logic.

Fix: Either (a) save each segment's scaler in seg_data, or (b) use the passed scaler when fit=False.

🔴 2. IQR Outlier Clipping Uses Future Data CRITICAL

Your IQR bounds are computed on df[NUMERIC].iloc[:train_end_iqr] which is correct for training. But then you apply df[NUMERIC].clip() to the entire dataframe including validation and test portions. The clip boundaries themselves leak training-set distributional info into test data.

Why it matters: This is a subtle but real leakage. Extreme outliers in the test set that would have been unseen by the model are silently capped to training-distribution bounds, making the model appear to handle anomalies better than it actually does.

Fix: This is borderline acceptable (clipping to train stats is a common approach). But be aware it can slightly inflate test metrics. The bigger problem is the interpolate().ffill().bfill() that follows — it uses future values to fill NaNs across the entire time series.

# The interpolate line leaks future into past:
df[NUMERIC] = df[NUMERIC].interpolate(method='linear').ffill().bfill()
# Fix: only forward-fill, never backward-fill across train/val/test
df[NUMERIC] = df[NUMERIC].ffill().fillna(0)

⚡ 6. Add Temporal Attention over LSTM Output HIGH

You only use the last hidden state (out[:, -1, :]). This discards 59 out of 60 timestep representations. For QoE forecasting, critical degradation patterns (brief spikes in jitter, short packet loss bursts) can appear at any position in the window.

Fix: Add a simple attention layer that learns which timesteps matter most:

class TemporalAttention(nn.Module):
    def __init__(self, hidden_size):
        super().__init__()
        self.attn = nn.Linear(hidden_size, 1)

    def forward(self, lstm_out):
        # lstm_out: (batch, seq_len, hidden)
        scores = self.attn(lstm_out).squeeze(-1)    # (batch, seq_len)
        weights = torch.softmax(scores, dim=1)      # (batch, seq_len)
        context = (lstm_out * weights.unsqueeze(-1)).sum(dim=1)
        return context  # (batch, hidden)

Then replace last = out[:, -1, :] with last = self.attention(out).

This is a bigger change, so treat it as a v2 experiment rather than an immediate fix.

⚙️ 14. Focal Gamma Too Low HIGH

You're using focal_gamma=1.0, which is almost standard cross-entropy. The original Focal Loss paper (Lin et al. 2017) recommends gamma=2.0 as the default, and your problem has significant class imbalance (3.9x ratio).

With gamma=1.0, the focal modulation factor (1-pt)^1 only provides mild down-weighting of easy examples. With gamma=2.0, a correctly classified sample with pt=0.9 gets weight 0.01 vs 0.1 — a 10x stronger focus on hard examples.

⚙️ 17. Add Mixup Augmentation MEDIUM

Time-series Mixup is a simple but effective regularizer. Interpolate between two random training sequences and their labels to create synthetic training data. This reduces overfitting and improves decision boundaries:

def mixup_data(x, y, alpha=0.2):
    lam = np.random.beta(alpha, alpha)
    idx = torch.randperm(x.size(0))
    mixed_x = lam * x + (1 - lam) * x[idx]
    y_a, y_b = y, y[idx]
    return mixed_x, y_a, y_b, lam

def mixup_criterion(criterion, pred, y_a, y_b, lam):
    return lam * criterion(pred, y_a) + (1-lam) * criterion(pred, y_b)

📝 Complete Improved Model Class

class LSTMForecasterV2(nn.Module):
    """LSTM with temporal attention for QoE forecasting."""

    def __init__(self, input_size, hidden_size, num_layers,
                 num_classes, dropout):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0.0,
        )
        self.layer_norm = nn.LayerNorm(hidden_size)

        # Temporal attention — learns which timesteps matter
        self.attn_proj = nn.Linear(hidden_size, 1)

        self.classifier = nn.Sequential(
            nn.Linear(hidden_size, hidden_size),
            nn.GELU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_size, hidden_size // 2),
            nn.GELU(),
            nn.Dropout(dropout * 0.5),
            nn.Linear(hidden_size // 2, num_classes),
        )

    def forward(self, x):
        out, _ = self.lstm(x)  # (B, T, H)
        out = self.layer_norm(out)

        # Attention-weighted pooling instead of last-only
        scores = self.attn_proj(out).squeeze(-1)  # (B, T)
        weights = torch.softmax(scores, dim=1)    # (B, T)
        ctx = (out * weights.unsqueeze(-1)).sum(1) # (B, H)

        return self.classifier(ctx)
